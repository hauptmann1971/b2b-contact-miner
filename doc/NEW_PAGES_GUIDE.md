# 🆕 Новые страницы веб-интерфейса

## Health Check страница (`/health-check`)

### Что это?
Красивая визуальная страница для мониторинга здоровья системы в реальном времени.

### Как использовать?
1. Откройте http://localhost:5000/health-check
2. Или нажмите кнопку "💚 Health Check" на главной странице

### Возможности:
- **Общий статус системы** - большой индикатор вверху страницы
- **Три типа проверок**:
  - 🔍 Полная проверка (БД + внутренние сервисы)
  - ❤️ Проверка живости (приложение запущено?)
  - ✅ Проверка готовности (БД доступна?)
- **Автообновление** - включите чекбокс для обновления каждые 10 секунд
- **Ручное обновление** - кнопка "🔄 Обновить" для каждой проверки
- **Цветовая индикация**:
  - 🟢 Зеленый = всё хорошо
  - 🔴 Красный = проблема
  - 🟡 Желтый = ожидание

## API Documentation страница (`/api-docs`)

### Что это?
Полная документация всех API эндпоинтов с примерами и возможностью тестирования.

### Как использовать?
1. Откройте http://localhost:5000/api-docs
2. Или нажмите кнопку "📡 API Docs" на главной странице

### Возможности:
- **Health Check API**:
  - `/health` - полная проверка
  - `/health/live` - проверка живости
  - `/health/ready` - проверка готовности
  
- **Data API**:
  - `/api/stats` - статистика системы
  - `/api/keywords` - список ключевых слов

- **Интерактивное тестирование**:
  - Нажмите "Загрузить данные" для получения реальной статистики
  - Данные отображаются в красивом формате с графиками
  - Полный JSON ответ показывается ниже

- **Примеры кода** для:
  - JavaScript (fetch)
  - Python (requests)
  - curl

---

## Навигация

Все страницы связаны между собой:
- С любой страницы можно вернуться на главную (кнопка "← На главную")
- С главной страницы есть ссылки на все разделы
- Health Check страница содержит быстрые ссылки на API эндпоинты

## Примеры использования

### Мониторинг системы
```
1. Откройте /health-check
2. Включите автообновление
3. Следите за статусом БД и сервисов мониторинга
```

### Тестирование API
```
1. Откройте /api-docs
2. Нажмите "Загрузить данные" для /api/stats
3. Увидите красивую визуализацию статистики
4. Ниже будет полный JSON ответ
```

### Быстрая проверка
```bash
# Из терминала
curl http://localhost:5000/health/live
curl http://localhost:5000/api/stats
```
