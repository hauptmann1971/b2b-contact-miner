# 🌐 Flask Web Server - B2B Contact Miner

Веб-интерфейс для управления ключевыми словами, просмотра результатов и мониторинга системы.

## 🚀 Запуск сервера

### 1. Установите зависимости:
```bash
pip install flask==3.0.0
```

### 2. Запустите веб-сервер:
```bash
python web_server.py
```

Сервер запустится на `http://localhost:5000`

## 📋 Доступные страницы

### Главная страница (`/`)
- **Статистика системы**: общее количество ключевых слов, доменов, контактов
- **Форма добавления ключевых слов**: быстрое добавление новых keywords
- **Последние ключевые слова**: таблица с недавно добавленными keywords
- **Навигация**: ссылки на все разделы, включая Health Check и API Docs

### Health Check страница (`/health-check`)
- **Визуальный интерфейс** для мониторинга здоровья системы
- **Три типа проверок**:
  - Полная проверка здоровья (БД + Redis)
  - Проверка живости (Liveness)
  - Проверка готовности (Readiness)
- **Автообновление** каждые 10 секунд (опционально)
- **Цветовая индикация** статусов (зеленый/красный/желтый)
- **Быстрый доступ** к API эндпоинтам

### API Documentation страница (`/api-docs`)
- **Полная документация** всех API эндпоинтов
- **Примеры ответов** в JSON формате
- **Интерактивные кнопки** для тестирования API
- **Визуализация данных** статистики в реальном времени
- **Примеры кода** для JavaScript, Python и curl

### Список ключевых слов (`/keywords`)
- Полная таблица всех ключевых слов
- Пагинация (20 записей на страницу)
- Фильтрация по статусу обработки
- Кнопки действий: просмотр деталей, удаление

### Детали ключевого слова (`/keyword/<id>`)
- Подробная информация о keyword
- Список найденных доменов
- Все найденные контакты с иконками типов
- Метаданные сайтов

### Список контактов (`/contacts`)
- Таблица всех найденных контактов
- Фильтры по типам: Email, Telegram, LinkedIn, Телефон
- Пагинация (50 записей на страницу)
- Ссылки на связанные keywords

## 🔍 Health Check Endpoints

### Полная проверка здоровья (`/health`)
Возвращает статус всех сервисов:
```json
{
  "status": "healthy",
  "timestamp": "2026-04-10T15:30:00",
  "services": {
    "database": "healthy",
    "redis": "healthy"
  }
}
```

### Проверка живости (`/health/live`)
Быстрая проверка, что приложение работает:
```json
{
  "status": "alive",
  "timestamp": "2026-04-10T15:30:00"
}
```

### Проверка готовности (`/health/ready`)
Проверка подключения к базе данных:
```json
{
  "status": "ready",
  "timestamp": "2026-04-10T15:30:00"
}
```

## 📡 API Endpoints

### Статистика (`/api/stats`)
Возвращает статистику системы в JSON формате:
```json
{
  "total_keywords": 15,
  "processed_keywords": 10,
  "total_domains": 45,
  "total_contacts": 120,
  "contacts_by_type": {
    "email": 80,
    "telegram": 20,
    "linkedin": 15,
    "phone": 5
  }
}
```

### Список ключевых слов (`/api/keywords`)
Возвращает массив всех keywords:
```json
[
  {
    "id": 1,
    "keyword": "финтех стартап",
    "language": "ru",
    "country": "RU",
    "is_processed": true,
    "last_crawled_at": "2026-04-10T15:30:00",
    "created_at": "2026-04-09T10:00:00"
  }
]
```

## 💡 Использование

### Добавление ключевого слова через форму:
1. Откройте `http://localhost:5000`
2. Введите ключевое слово в форму
3. Выберите язык и страну
4. Нажмите "Добавить ключевое слово"

### Просмотр результатов:
1. Перейдите на страницу "Все ключевые слова"
2. Нажмите "Просмотр" рядом с нужным keyword
3. Увидите все найденные домены и контакты

### Фильтрация контактов:
1. Перейдите на страницу "Все контакты"
2. Используйте кнопки фильтров для выбора типа
3. Просматривайте отфильтрованные результаты

## 🎨 Особенности интерфейса

- **Современный дизайн**: градиенты, тени, анимации
- **Адаптивная верстка**: работает на мобильных устройствах
- **Цветовая индикация**: разные цвета для разных типов контактов
- **Уведомления**: flash-сообщения об успехе/ошибках
- **Пагинация**: удобная навигация по большим спискам

## 🔧 Конфигурация

Для изменения порта или хоста отредактируйте конец файла `web_server.py`:

```python
app.run(
    host='0.0.0.0',  # Слушать все интерфейсы
    port=5000,        # Порт
    debug=True        # Режим отладки
)
```

## 📊 Интеграция с базой данных

Веб-сервер использует ту же базу данных MySQL, что и основной пайплайн. Все изменения отображаются в реальном времени.

## ⚠️ Примечания

- Для production использования отключите `debug=True`
- Настройте `SECRET_KEY` в `.env` файле для безопасности сессий
- Рекомендуется использовать reverse proxy (nginx) для production
