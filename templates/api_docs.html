🔍 Health Check Endpoints

Полная проверка здоровья GET
/health

Возвращает статус всех сервисов системы (база данных, Redis и др.)

Пример ответа:
{
  "status": "healthy",
  "timestamp": "2026-04-10T15:30:00",
  "services": {
    "database": "healthy",
    "redis": "healthy"
  }
}
Попробовать →
Проверка живости GET
/health/live

Быстрая проверка, что приложение запущено и отвечает на запросы

Пример ответа:
{
  "status": "alive",
  "timestamp": "2026-04-10T15:30:00"
}
Попробовать →
Проверка готовности GET
/health/ready

Проверяет подключение к базе данных и готовность обрабатывать запросы

Пример ответа:
{
  "status": "ready",
  "timestamp": "2026-04-10T15:30:00"
}

📊 Data API Endpoints

Статистика системы GET
/api/stats

Возвращает полную статистику по ключевым словам, доменам и контактам с разбивкой по типам

Пример ответа:
{
  "total_keywords": 15,
  "processed_keywords": 10,
  "total_domains": 45,
  "total_contacts": 120,
  "contacts_by_type": {
    "email": 80,
    "telegram": 20,
    "linkedin": 15,
    "phone": 5
  }
}
Список ключевых слов GET
/api/keywords

Возвращает массив всех ключевых слов с полной информацией о каждом

Пример ответа:
[
  {
    "id": 1,
    "keyword": "финтех стартап",
    "language": "ru",
    "country": "RU",
    "is_processed": true,
    "last_crawled_at": "2026-04-10T15:30:00",
    "created_at": "2026-04-09T10:00:00"
  }
]

💡 Использование API

Из JavaScript:
fetch('/api/stats').then(r => r.json()).then(data => console.log(data))

Из Python:
import requests
response = requests.get('http://localhost:5000/api/stats')

Из curl:
curl http://localhost:5000/api/stats
